01 Visi, Misi & Tujuan Produk

Visi

Menjadi sistem operasi TV open-source paling ringan, aman, dan developer-friendly untuk perangkat ARM — dari TV Box konsumen hingga panel signage komersial.

Misi

  • Menyediakan pengalaman TV cerdas lengkap dengan UI berbasis web (HTML/CSS/JS) seperti webOS, namun dengan footprint RAM < 256 MB saat idle.
  • Menggunakan Rust untuk seluruh system services — menjamin memory safety dan zero-cost abstractions tanpa garbage collector.
  • Mendukung OTA update dual-slot A/B yang atomik dan rollback-safe, cocok untuk deployment di perangkat konsumen jarak jauh.
  • Dapat di-build dan dijalankan di QEMU (aarch64) dalam satu perintah — memangkas feedback loop pengembangan.

Kenapa Rust?

🛡️ Memory Safety

Tanpa GC. Tanpa use-after-free. Tanpa data race. Compiler menjamin safety di compile-time — ideal untuk system services yang long-running.

⚡ Zero-Cost Abstractions

Async runtime (tokio), serialization (serde), IPC — semua zero-cost. Performa setara C tanpa risiko.

📦 Cargo Ecosystem

Smithay (Wayland), pipewire-rs, dbus-rs, rauc-rs, netlink-rs — ekosistem Rust siap pakai untuk komponen OS.

🔒 Supply Chain

cargo vet + cargo audit + cargo deny — rantai pasok aman, cocok untuk OTA production.

03 Kompositor & Web Runtime

Lumin Compositor

Ditulis dalam Rust menggunakan Smithay — library Wayland compositor modular. Lumin bertindak sebagai system compositor yang menjalankan web runtime (WPE WebKit) sebagai client Wayland tunggal.

Fitur Kompositor

  • Wayland Server — kompatibel penuh dengan protokol Wayland 1.22+ (xdg-shell, wlr-layer-shell, wp-presentation-time, linux-dmabuf, zwp_linux_dmabuf_v1)
  • Hardware Overlay Planner — memanfaatkan KMS planes hardware untuk video playback zero-copy (overlay plane terpisah dari primary/RGB plane)
  • Scene Graph API — API internal untuk animasi UI (transisi 60fps menggunakan timer page-flip DRM + damage tracking)
  • Multi-layer compositing — Background → Video → UI → Cursor → OSD
  • HDR Support — HDR10/HLG via KMS HDR properties (Colorspace, HDR_OUTPUT_METADATA) pada kernel ≥6.6
  • DRM Lease — opsional: lease output ke client eksternal (mis. Kodi, mpv) untuk playback tanpa compositing overhead

Web Runtime: WPE WebKit

WPE WebKit (Web Platform for Embedded) adalah port WebKit yang dioptimalkan untuk perangkat embedded. Tidak memerlukan toolkit GUI — langsung render ke EGL/GLES surface yang disediakan kompositor via Wayland.

Custom WPE Backend

Kami menulis backend WPE kustom (libwpebackend-uos) dalam Rust (via FFI binding ke C API WPE). Backend ini:

  • Mendaftarkan surface WebKit sebagai wl_surface ke Lumin compositor
  • Mengirimkan input event dari compositor ke WebKit
  • Memanfaatkan linux-dmabuf untuk zero-copy buffer sharing
  • Mengekspos native HTML5 video pipeline via GStreamer + V4L2 (HW decode)
  • DRM (Widevine L3) playback via platform DRM integration

🎬 HW Video Decode

V4L2 stateless decoder (H.264/H.265/VP9/AV1) → dmabuf → zero-copy ke KMS overlay plane. Tidak ada salinan memori tambahan sepanjang pipeline video.

🎮 WebGL 2.0

ANGLE backend OpenGL ES 3.2 native. Performa GPU penuh untuk game dan animasi WebGL.

Contoh: Stardust Message Client (Rust)

// stardust-client.rs — contoh publish dari service
use stardust::{Client, Message, Method};

#[tokio::main]
async fn main() -> stardust::Result<()> {
    let client = Client::connect("/run/uos/bus.sock").await?;

    // Subscribe ke event network
    let mut rx = client.subscribe("network.*").await?;

    // Publish status
    client.publish(Message::new("network.status")
        .param("connected", true)
        .param("iface", "eth0")
    ).await?;

    // Request-response
    let resp = client.call("ota.check_update", ())
        .timeout(Duration::from_secs(5))
        .await??;

    Ok(())
}

05 OTA Update System

Sistem OTA UOS TV menggunakan dual-slot A/B update dengan RAUC (Robust Auto-Update Controller) sebagai framework update, dan casync untuk delta-based image delivery yang hemat bandwidth.

Arsitektur Dual-Slot A/B

┌───────────────────────────────────────────────────┐ │ eMMC / NVMe (8GB+) │ │ │ │ ┌─────────┐ ┌──────────┐ ┌──────┐ ┌───────────┐ │ │ │ Slot A │ │ Slot B │ │ Data │ │ Recovery │ │ │ │ (boot) │ │ (update) │ │ │ │ (factory) │ │ │ │ 2.5 GB │ │ 2.5 GB │ │ 2 GB │ │ 1 GB │ │ │ └────┬────┘ └────┬─────┘ └──────┘ └───────────┘ │ │ │ │ │ │ └───┬───────┘ │ │ │ │ │ ┌────────▼───────┐ │ │ │ U-Boot SPL │ → memilih slot via bootcount │ │ │ + environment │ & BOOT_ORDER │ │ └────────────────┘ │ └───────────────────────────────────────────────────┘

Proses Update OTA — Step by Step

  1. Polling / Push Notification — otad memeriksa update server via HTTPS secara periodik (default: setiap 6 jam) atau menerima push notification via WebSocket persistent connection.
  2. Metadata Download — Unduh update.yaml berisi versi, ukuran, hash SHA256, changelog, dan flag critical.
  3. Delta Computation — Server menyediakan casync index (.caibx). otad menggunakan casync-rs untuk menghitung chunk yang sudah ada (reuse dari local store atau slot B).
  4. Download Chunks — Hanya chunk yang berubah diunduh. Estimasi bandwidth: 30-80% lebih kecil dari full image.
  5. Verification — Hash SHA256 setiap chunk diverifikasi saat diterima. Bundle di-assemble dengan streaming verification.
  6. Install ke Slot B — Image di-write ke partisi slot B menggunakan rauc. Partisi slot A tidak disentuh selama proses ini.
  7. Bootloader Update — U-Boot environment variable: BOOT_ORDER di-set ke "B A", BOOT_B_LEFT = 3 (try B, fallback A).
  8. Reboot — Sistem reboot ke slot B. Jika boot berhasil, otad menandai slot B sebagai good via rauc status mark-good.
  9. Rollback — Jika boot gagal (kernel panic, watchdog, atau service critical crash), U-Boot akan decrement counter dan fallback ke slot A. Data partisi tetap aman.

Komponen OTA — Rust Crates

CrateFungsi
otadService utama OTA: polling, download, install orchestration
rauc-rsRust binding untuk RAUC D-Bus API (check, install, mark)
casync-rsRust implementasi casync chunking + reassembly (pure Rust, no C)
update-verifyVerifikasi signature (Ed25519) dan hash chain bundle
ota-configKonfigurasi server URL, interval, retry policy, metered network policy

Server-Side (Update Backend)

Update server adalah service Rust (axum + SQLite) yang menyediakan:

  • REST API — GET /api/v1/update/check?device=...&current=... mengembalikan metadata update
  • Chunked Download — GET /api/v1/update/download/:bundle_id dengan dukungan range request + resume
  • casync Index Generation — Server otomatis membuat .caibx index setiap kali build baru diunggah
  • Release Channels — stable, beta, dev. Setiap perangkat bisa memilih channel.
  • Staged Rollout — Persentase rollout bertahap (10% → 50% → 100%) dengan automatic halt jika error rate meningkat.

Keamanan OTA

  • Ed25519 Signature — Bundle update ditandatangani dengan Ed25519. Public key di-embed di kernel command line / dtb dan diverifikasi oleh RAUC saat install.
  • TLS 1.3 + Certificate Pinning — Semua komunikasi HTTPS dengan rustls + pinning certificate hash.
  • dm-verity — Partisi rootfs menggunakan dm-verity. Hash tree diverifikasi saat read, mencegah modifikasi offline.
  • Hardware Root of Trust — Jika tersedia TEE (OP-TEE / TrustZone), kunci signature disimpan di secure storage. Jika tidak, fallback ke efuse/OTP area.

07 Boot Process

1. ROM Bootloader → U-Boot SPL

SoC ROM memuat U-Boot SPL dari eMMC/SD. SPL menginisialisasi DRAM, memeriksa bootcount untuk A/B slot selection, lalu memuat U-Boot proper.

2. U-Boot Proper

U-Boot membaca extlinux.conf dari partisi boot slot terpilih. Memuat kernel Image.gz, DTB, dan initramfs. Meneruskan dm-mod.create=... untuk dm-verity rootfs.

3. Linux Kernel Boot

Kernel inisialisasi driver, mount rootfs SquashFS via dm-verity, pivot_root, lalu menjalankan /init dari initramfs.

4. init (inis) — Tiny Init (Rust)

inis adalah init system minimalis dalam Rust: mount filesystem (/data, /run, /tmp), setup loopback, start monitord sebagai PID 2.

5. monitord — Process Supervisor

monitord membaca service manifest dari /usr/share/uos/services.d/, lalu start semua service sesuai urutan dependensi (dependency graph).

6. Service Startup Order

① logd → ② devmand → ③ dispald → ④ inputd → ⑤ lumind (compositor) → ⑥ audiod → ⑦ netmd → ⑧ notifd → ⑨ powermand → ⑩ pkgd → ⑪ otad

7. Luna UI Shell Launch

lumind menjalankan WPE WebKit yang memuat file:///usr/share/luna/index.html. Luna shell menampilkan boot animation → home launcher.

8. Ready

Sistem siap. otad memulai polling OTA di background. Total boot time target: < 8 detik (cold boot, ke home launcher).

inis — Tiny Init (Rust)

// inis/src/main.rs — init system minimalis
use nix::mount::{mount, MsFlags};
use nix::unistd::{chdir, chroot, execve};
use std::process::Command;

fn main() {
    // Mount pseudo-filesystems
    mount(Some("devtmpfs"), "/dev", "devtmpfs", MsFlags::empty(), None::&str).unwrap();
    mount(Some("proc"), "/proc", "proc", MsFlags::empty(), None::&str).unwrap();
    mount(Some("sysfs"), "/sys", "sysfs", MsFlags::empty(), None::&str).unwrap();
    mount(Some("tmpfs"), "/run", "tmpfs", MsFlags::empty(), None::&str).unwrap();

    // Mount data partition
    mount(Some("/dev/mmcblk0p5"), "/data", "ext4", MsFlags::empty(), None::&str).unwrap();

    // Start process supervisor as PID 2
    let monitord = Command::new("/usr/bin/monitord")
        .spawn()
        .expect("Failed to start monitord");

    // Reap zombies, wait forever
    loop {
        nix::sys::wait::wait().unwrap();
    }
}

10 Roadmap & Timeline Pengembangan

Fase 1 — Foundation (Bulan 1-3)

✅ Armbian Base Bring-Up

Build Armbian rootfs untuk target QEMU virt aarch64. Kernel 6.6 bootable di QEMU. GPU VirGL berfungsi penuh.

✅ inis + monitord

Init system minimalis + process supervisor dalam Rust. Dependency graph service ordering.

✅ stardust IPC

Library IPC pub/sub + broker daemon. Unix socket + CBOR. Request-response + event streaming.

✅ logd

Centralized logging dengan tracing crate. Structured JSON log output.

Fase 2 — Display & Input (Bulan 4-6)

🎯 lumind — Basic Wayland Compositor

Smithay-based DRM/KMS compositor. Bisa menjalankan weston-terminal dan simple EGL client via VirGL di QEMU.

🎯 dispald + inputd

DRM display management (EDID, modeset). evdev input handling (keyboard, mouse, touch, remote control).

🎯 WPE WebKit Integration

Build WPE WebKit untuk aarch64. Custom backend libwpebackend-uos. WebKit surface muncul sebagai client Wayland lumind.

Fase 3 — UI & Apps (Bulan 7-9)

🎯 Luna Shell v1

Home launcher dengan card-based UI, settings panel, app launching. Animasi transisi 60fps CSS GPU-accelerated.

🎯 audiod + netmd

PipeWire session manager, audio routing, volume control. WiFi/Ethernet network management dengan UI.

🎯 pkgd — App Manager

Install/uninstall aplikasi web (PWA-style). Per-app sandbox (separate WPE WebView process).

🎯 Hardware Video Decode

V4L2 stateless decode pipeline terintegrasi ke WPE WebKit HTML5 video. Zero-copy ke KMS overlay plane.

Fase 4 — OTA & Production (Bulan 10-12)

🎯 otad — OTA Manager

Dual-slot A/B update dengan RAUC + casync. Delta update, automatic rollback, boot counting.

🎯 Update Server

Rust backend (axum) untuk OTA delivery. Release channels, staged rollout, dashboard monitoring.

🎯 Security Hardening

dm-verity rootfs, SELinux enforcing, capability dropping per service, seccomp-bpf syscall filtering, TLS certificate pinning.

🎯 Hardware Bring-Up

Porting ke hardware nyata: Rockchip RK3566/RK3588, Amlogic S905X. U-Boot board config, DTB, driver enablement.

Fase 5 — Ecosystem (Bulan 13+)

  • App Store — Web app marketplace dengan rating, review, dan automated security review (Content Security Policy audit)
  • SDK & Simulator — Rust SDK untuk app development + QEMU simulator dengan hot-reload
  • Voice Control — On-device speech recognition (whisper.cpp via Rust binding) atau cloud-based
  • Multi-display — Dukungan dual/triple display untuk signage
  • HDMI-CEC Deep Integration — Kontrol TV penuh via CEC (power, volume, input switching)

11 Risiko & Mitigasi

RisikoSeverityMitigasi
GPU driver open-source (Panfrost/Lima) tidak stabil di SoC tertentu High Prioritaskan SoC dengan dukungan GPU matur (RK3566/RK3588). QEMU VirGL sebagai fallback development target.
VPU (video decoder) tidak didukung V4L2 stateless API di semua SoC Medium Gunakan Hantro/RKVDEC yang sudah upstream. Untuk SoC tanpa dukungan, fallback ke software decode via FFmpeg (single 1080p H.264).
Smithay belum mature untuk production compositor Medium Kontribusi upstream ke Smithay untuk fitur yang dibutuhkan. Fallback: wlroots + wlroots-rs binding.
WPE WebKit build untuk aarch64 cross-compilation kompleks Medium Build di QEMU aarch64 native VM (meski lambat) atau gunakan Yocto/Buildroot toolchain yang sudah menyediakan WPE recipe.
RAUC memerlukan bootloader support (U-Boot integration) Low U-Boot sudah mendukung BOOTCOUNT + BOOT_ORDER untuk A/B. Uji awal di QEMU dengan U-Boot virt target.
Ekosistem Rust bare-metal embedded masih berkembang Low Kita tidak bare-metal — kita di atas Linux + Armbian. Ekosistem Rust untuk Linux userspace sudah sangat matur.
